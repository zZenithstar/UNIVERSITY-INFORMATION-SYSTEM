# UNIVERSITY-INFORMATION-SYSTEM
Java-based console School Management System with Admin, Teacher, and Student roles. Includes login, profile management, and CRUD operations using text-file storage.



A console-based School Management System developed in Java using Object-Oriented Programming principles. Built because apparently humanity enjoys managing chaos through menus.

---Features---
Multi-role login system (Admin / Teacher / Student)
Add, update, delete users
Profile viewing system
Text-file based data storage
Menu-driven console interface
Separate management for Students, Teachers, and Admins
Organized class structure with encapsulation

---Technologies Used---
Java
OOP Concepts (Encapsulation, Classes, Methods)
File I/O (BufferedReader, FileWriter, FileReader)
IntelliJ IDEA

---Project Purpose---

This project was created to improve Java programming skills, file handling knowledge, and Object-Oriented Programming logic through a real-world management system simulation.

-File System & Data Management-
This project uses a file-based system to store and manage data. To ensure the application runs correctly without any FileNotFoundException, please follow these instructions:

-File Locations-
The system expects the data files to be at the following relative path:

Path: src/ProjectMain/txts/

Files: Admins.txt, Students.txt, Teachers.txt

-Configuration for Execution-
Since the project uses Relative Paths, the Working Directory must be set to the Project Root (the main folder containing the src folder).

If using IntelliJ/Eclipse/VS Code: Open the project from the top-level folder. Do not open the src or ProjectMain folder individually.

Working Directory: Ensure your IDE's run configuration is set to $PROJECT_DIR$.

Note: The program is designed to find the database files automatically as long as the project structure remains intact.
